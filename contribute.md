# Contributing to PEITT523 Cloud Computing

Thank you for your interest in contributing to this course repository! This document provides guidelines for contributing improvements, corrections, and enhancements.

## How to Contribute

### Reporting Issues

If you find an error, broken link, or have a suggestion for improvement:

1. Check existing issues to avoid duplicates
2. Open a new issue with a clear description:
   - What is the problem or suggestion?
   - Where is it located (file and section)?
   - Why is this important?

### Suggesting Improvements

- **Content improvements:** Suggest better explanations, additional examples, or clearer organization
- **Resource updates:** Recommend new relevant links or remove outdated ones
- **Structure enhancements:** Propose better ways to organize or present information

### Submitting Changes

1. **Fork the repository** and create a branch for your changes
2. **Make your edits:**
   - Keep markdown formatting consistent
   - Ensure links are valid and working
   - Test any new links before submitting
3. **Commit with clear messages** describing the change
4. **Submit a pull request** with:
   - A clear title describing the change
   - A brief explanation of what was changed and why
   - Reference to any related issues

## Content Guidelines

### Markdown Format

- Use consistent heading levels (# for main, ## for sections, ### for subsections)
- Keep line lengths reasonable for readability
- Use backticks for code, file names, and technical terms
- Add descriptive alt text for any images

### Resource Links

- Verify links are current and relevant
- Prioritize official documentation and reputable sources
- Include the link description/purpose
- Update or remove broken links

### Code Examples

- Include context and explanation
- Test code snippets if applicable
- Use proper syntax highlighting

## Code of Conduct

- Be respectful and professional in all interactions
- Focus on the content and ideas, not on individuals
- Provide constructive feedback
- Help other contributors succeed

## Attribution

- All contributions are welcome and appreciated
- Contributors will be recognized in significant changes
- This material is provided under the course's licensing terms

## Questions?

If you have questions about contributing, please open an issue or reach out to the course instructor.

---

**Happy contributing!** Your help makes this course better for all students.
